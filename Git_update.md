# Git: Как вносить изменения на GitHub

## Базовые команды (ежедневное использование)

### 1. Проверить статус изменений
```powershell
git status
```
Покажет какие файлы были изменены, добавлены или удалены.

### 2. Добавить изменения
```powershell
# Добавить все изменённые файлы
git add .

# Или добавить конкретный файл
git add путь/к/файлу.py
```

### 3. Создать коммит (сохранить изменения)
```powershell
git commit -m "Описание что было сделано"
```

### 4. Отправить на GitHub
```powershell
git push
```

---

## Быстрая команда "всё в одном"

Если хотите добавить все изменения и отправить одной строкой:
```powershell
git add . && git commit -m "Описание изменений" && git push
```

---

## Полезные команды

### Посмотреть историю коммитов
```powershell
git log --oneline
```

### Отменить изменения в файле (до коммита)
```powershell
git checkout -- путь/к/файлу.py
```

### Скачать изменения с GitHub (если работаете с нескольких компьютеров)
```powershell
git pull
```

### Посмотреть разницу в файлах
```powershell
git diff
```

## Примеры хороших сообщений коммитов

```
git commit -m "Добавлен новый endpoint для анализа"
git commit -m "Исправлена ошибка подключения к базе данных"
git commit -m "Обновлены зависимости в requirements.txt"
git commit -m "Рефакторинг функции транскрипции"
git commit -m "Добавлена документация к API"
```

---

## Типичный рабочий процесс

1. Внесли изменения в код
2. `git status` — проверили что изменилось
3. `git add .` — добавили изменения
4. `git commit -m "Описание"` — сохранили с комментарием
5. `git push` — отправили на GitHub

---

## Решение частых проблем

### Забыли что-то добавить в последний коммит
```powershell
git add .
git commit --amend -m "Новое описание коммита"
git push --force
```

### Конфликт при push (кто-то другой внёс изменения)
```powershell
git pull --rebase
git push
```

### Откатить последний коммит (сохранив изменения в файлах)
```powershell
git reset --soft HEAD~1
```
